Notes

You will notice a progressive experimentation. All these buttons except the first return the same results. There are several ways to call RESTful services implemented via Web Api.

It's not clear the advantage of GET vs POST vs PUT vs DELETE because with Attribute Based Routing we can do everything via GET or POST or PUT

Until we learn otherwise we will do everything via the get. It is likely we will always return something whether we are getting, updating, inserting, or deleting so GET seems to be the verb that would be most consistent.

Some components, such as response messages might not be implemented yet. These methods will be massaged and improved as we learn more.
